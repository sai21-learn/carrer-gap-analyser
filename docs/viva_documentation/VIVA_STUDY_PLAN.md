## 🗣️ OPENING STATEMENT (First 2 minutes)

When viva starts, you might hear: *"Tell us about your project."*

**Use this structure (memorized):**

"Good morning. My project is an AI Skill Gap Analyzer. 

The problem it solves: College students often don't know what specific skills a job market wants for their target role. Manual research is slow and subjective.

My solution: A Streamlit web application that automatically analyzes real job postings, compares them to student skills, and recommends personalized learning resources.

The system is a 5-stage pipeline:
1. Student enters name, role, and current skills via web form
2. Job scraper fetches postings from LinkedIn, Naukri, and Adzuna
3. NLP engine extracts and normalizes required skills
4. Gap analyzer classifies each skill as Matched, Partial, or Gap
5. Recommender maps gaps to curated learning resources

The output is an interactive dashboard showing match score and actionable recommendations.

Key technical highlights:
- Python with Streamlit for rapid UI development
- spaCy for NLP skill extraction
- Modular architecture (5 independent services)
- 50+ test cases with 80% coverage
- Fallback to mock data (resilience design)

That's the 30,000-foot overview. Happy to dive deeper into any aspect."

**Time: ~2 minutes** | **Confidence: High** | **Clarity: Crystal clear**

Practice saying this without notes until it's smooth!

## 🎯 TACTICAL TIPS FOR ANSWERING

### When they ask a question:

1. **Pause (2 seconds)** - Formulate your answer
2. **State main idea** - "The answer is: [clear statement]"
3. **Explain why** - "The reasoning is: [context]"
4. **Give example** - "For instance, with Data Analyst role..."
5. **Check understanding** - "Does that clarify it?" or wait for follow-up

### If you don't know:

**DON'T:** "I haven't thought about that" or stare blankly
**DO:** "That's a great question. Here's how I'd approach it: [think through logic]. In the actual implementation, I chose [what you did] because [rationale]."

This shows:
- Critical thinking ✓
- Humble confidence ✓
- Awareness of limitations ✓
- Problem-solving ability ✓

### If they push back:

Example: *"Why 0.7 threshold? Seems arbitrary."*

**DON'T:** "It's just what works" (weak)
**DO:** "It's empirically tuned. We tested on 5 sample jobs and found 0.7 balances precision/recall. Below 0.7, false positives (e.g., Git ≈ GitHub). Above 0.7, false negatives (e.g., ML ≉ Machine Learning). Configurable if different data suggests otherwise."

This shows:
- Intentional design ✓
- Testing mindset ✓
- Flexibility ✓
- Confidence in trade-off reasoning ✓
